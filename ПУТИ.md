# Пути к папкам — что куда класть

## Как проверить, что всё готово

**Сначала перейдите в папку проекта.** В командной строке (cmd) или PowerShell:

```cmd
cd "%USERPROFILE%\OneDrive\Рабочий стол\NEW PROJECT\tasks-ui"
```

(Если проект на другом диске или в другой папке — укажите свой путь, например `cd D:\NEW PROJECT\tasks-ui`.)

| Шаг | Действие | Ожидаемый результат |
|-----|----------|----------------------|
| 1 | В папке `tasks-ui` выполнить `npm install` | Установка без ошибок, появилась папка `node_modules` |
| 2 | Скопировать `.env.example` в `.env` | В `tasks-ui` есть файл `.env` |
| 3 | В папке `tasks-ui` выполнить `npm start` | В консоли: «Сервер: http://localhost:3080» |
| 4 | Открыть в браузере http://localhost:3080 | Страница «Задачи и напоминания» открылась |
| 5 | Посмотреть внизу страницы | Зелёный текст: «Сервер подключён…» |
| 6 | Добавить задачу и нажать «Добавить задачу» | Появился тост «Сохранено», задача в списке |
| 7 | Проверить папку `tasks-ui/data/` | Есть файл `tasks.json` |

**Готово:** если шаги 1–6 выполнены — интерфейс и сервер работают.  
**Уведомления в Telegram:** нужен запущенный OpenClaw Gateway и заполненный `.env` (токен, Chat ID); затем создать задачу с напоминанием через пару минут и дождаться сообщения в Telegram.

## Папка tasks-ui (интерфейс + уведомления)

| Путь | Назначение |
|------|------------|
| **tasks-ui/** | Корень. Здесь в терминале выполнять `npm install` и `npm start`. |
| **tasks-ui/index.html** | Страница в браузере. Не переносить в подпапки. |
| **tasks-ui/server.js** | Сервер и планировщик. Не переносить. |
| **tasks-ui/package.json** | Зависимости (npm). Не переносить. |
| **tasks-ui/.env.example** | Образец настроек. Скопировать в `.env`. |
| **tasks-ui/.env** | Создать вручную. Сюда писать токены и Telegram Chat ID. В репозиторий не коммитить. |
| **tasks-ui/data/** | Создаётся сама при первом сохранении задачи. Вручную не создавать. |
| **tasks-ui/data/tasks.json** | Список задач. Пишет сервер. Можно подложить свой JSON при импорте. |
| **tasks-ui/data/.reminder-state.json** | Служебный файл планировщика. Не редактировать. |

---

## Что куда класть вы сами

- **.env** — создать в папке **tasks-ui** (скопировать из `.env.example` и заполнить).
- **Резервная копия задач** — сохранять файл **tasks-ui/data/tasks.json** куда удобно (бэкап).
- **Весь проект** — папку **NEW PROJECT** (вместе с **tasks-ui**) можно положить куда угодно: Рабочий стол, `C:\Projects`, диск D: и т.д.

---

## Чего не делать

- Не переносить **index.html**, **server.js**, **package.json** во вложенные папки — сервер ищет их в корне **tasks-ui**.
- Не удалять папку **data** — в ней хранятся задачи и состояние напоминаний.
- Не коммитить **.env** в git — в нём секреты (токены, Chat ID).
