# Задачи и напоминания — готовый веб-проект

Один запуск — веб-интерфейс, сохранение задач на сервере и **уведомления в Telegram** по расписанию (через OpenClaw).

## Быстрый старт

**Перейти в папку проекта (Windows):**

```cmd
cd "%USERPROFILE%\OneDrive\Рабочий стол\NEW PROJECT\tasks-ui"
```

Если проект лежит в другом месте — подставьте свой путь, например:
```cmd
cd D:\Мои проекты\NEW PROJECT\tasks-ui
```

**Дальше:**

```bash
npm install
```

Создайте файл `.env` в папке `tasks-ui` (скопируйте из `.env.example` и при необходимости отредактируйте). В PowerShell можно так:
```powershell
copy .env.example .env
```

Запуск:
```bash
npm start
```

Откройте в браузере: **http://localhost:3080**

- Добавляйте задачи (дата, время, напомнить заранее, повтор).
- Указывайте получателя в Telegram (Chat ID) в форме или в каждой задаче.
- Задачи сохраняются на сервере в `data/tasks.json`.
- Раз в минуту сервер проверяет расписание и отправляет напоминания в Telegram через OpenClaw.

## Как проверить, что всё готово

1. **Установка**
   - В папке `tasks-ui` выполнено: `npm install` (без ошибок).
   - Есть файл `.env` (скопирован из `.env.example` и при необходимости заполнен).

2. **Запуск**
   - В терминале из папки `tasks-ui`: `npm start`.
   - В консоли есть строка вида: `Сервер: http://localhost:3080`.

3. **Интерфейс**
   - В браузере открыть http://localhost:3080.
   - Внизу страницы зелёный статус: **«Сервер подключён. Напоминания отправляются в Telegram по расписанию.»**
   - Если красный текст «Нет связи с сервером» — сервер не запущен или порт занят.

4. **Сохранение задач**
   - Добавить любую задачу (название, дата/время) и нажать «Добавить задачу».
   - Должен появиться тост **«Сохранено»** и задача в списке.
   - В папке `tasks-ui/data/` появился файл `tasks.json`.

5. **Уведомления в Telegram** (если настроен OpenClaw)
   - OpenClaw Gateway запущен, в `.env` указаны `OPENCLAW_HOOK_TOKEN` и `OPENCLAW_TELEGRAM_TO`.
   - Создать задачу с напоминанием **«В момент времени»** на время через 2–3 минуты.
   - Подождать 1–2 минуты после этого времени — в Telegram должно прийти сообщение от бота.

**Кратко:** зелёный статус внизу страницы + тост «Сохранено» при добавлении задачи = интерфейс и сервер готовы. Приход сообщения в Telegram = цепочка до уведомлений работает.

## Что нужно для уведомлений в Telegram

1. **OpenClaw Gateway** запущен (например `openclaw gateway --port 18789`), с включёнными hooks и каналом Telegram.
2. В **.env** заданы:
   - `OPENCLAW_GATEWAY_URL` — URL Gateway (по умолчанию `http://127.0.0.1:18789`).
   - `OPENCLAW_HOOK_TOKEN` — токен для `POST /hooks/agent`.
   - `OPENCLAW_TELEGRAM_TO` — ваш Telegram Chat ID (или несколько через запятую).

Без этих переменных интерфейс и список задач работают, но уведомления в Telegram отправляться не будут (в консоли сервера будет предупреждение).

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `PORT` | Порт веб-сервера (по умолчанию 3080) |
| `TZ` | Часовой пояс процессов (`Europe/Moscow` для МСК) |
| `OPENCLAW_GATEWAY_URL` | URL OpenClaw Gateway |
| `OPENCLAW_HOOK_TOKEN` | Токен webhook OpenClaw |
| `OPENCLAW_TELEGRAM_TO` | Telegram Chat ID получателя (несколько через запятую) |
| `CHECK_INTERVAL_MS` | Интервал проверки напоминаний в мс (по умолчанию 60000) |

## Пути: что куда класть

| Куда | Что класть |
|------|------------|
| **tasks-ui/** | Вся папка проекта. Открывайте эту папку в терминале и запускайте `npm start`. |
| **tasks-ui/.env** | Создайте файл сами (скопируйте из `.env.example`). Сюда: `OPENCLAW_HOOK_TOKEN`, `OPENCLAW_TELEGRAM_TO`, при необходимости `PORT`, `OPENCLAW_GATEWAY_URL`. |
| **tasks-ui/data/** | Папку создавать не нужно — появится после первого сохранения задачи в интерфейсе. |
| **tasks-ui/data/tasks.json** | Создаётся сервером. Можно вручную подложить готовый JSON с массивом `tasks` (как при экспорте из интерфейса), если хотите импорт «файлом». |
| Никуда не копировать | **index.html**, **server.js**, **package.json** — лежат в корне `tasks-ui/`, не переносите их в подпапки. |

**Куда ставить весь проект:** в любое место на диске, например  
`C:\Projects\tasks-ui` или `D:\NEW PROJECT\tasks-ui`. Путь может содержать кириллицу (Рабочий стол), но в терминале иногда проще работать из пути без пробелов и русских букв.

## API

- **GET /api/tasks** — вернуть список задач `{ "tasks": [...] }`.
- **POST /api/tasks** — тело `{ "tasks": [...] }` — сохранить задачи (полная перезапись).

## Формат задачи

- `id` — уникальный идентификатор.
- `title` — текст задачи.
- `dateTime` — дата/время в ISO (например `2025-03-15T10:00:00.000Z`).
- `remindBeforeMinutes` — за сколько минут напомнить (0, 5, 15, 30, 60, 1440, 2880).
- `repeat` — `once` | `daily` | `weekly` | `monthly`.
- `telegramTo` — необязательно; Telegram Chat ID (или несколько через запятую) для этой задачи.

Уведомления приходят в момент «дата/время минус напомнить заранее». Повторы считаются автоматически.

## Только интерфейс без сервера

Если открыть `index.html` в браузере напрямую (файлом), запросы к `/api/tasks` не сработают — внизу страницы будет сообщение «Нет связи с сервером». Запустите `npm start` и откройте http://localhost:3080.

## Запуск на сервере через PM2 (рекомендуется)

В проекте есть `ecosystem.config.cjs`, который поднимает:
- `assistent-web` (`server.js`)
- `assistent-bot` (`telegram-bot.js`)

1. Заполните `.env` на сервере (особенно `TELEGRAM_BOT_TOKEN`).
2. Выполните:

```bash
cd /opt/ASSISTENT1.0/tasks-ui
npm install
pm2 delete assistent-bot assistent-web || true
pkill -f telegram-bot.js || true
pm2 start ecosystem.config.cjs
pm2 save
pm2 ls
```

Логи:

```bash
pm2 logs assistent-bot --lines 50
pm2 logs assistent-web --lines 50
```

## Команды Telegram-бота

- `/start`, `/help` — подсказка
- `/time` — показать текущее время бота и его timezone
- `/menu` — показать меню с кнопками
- `/list` — список ваших задач
- `/delete N` — удалить задачу по номеру из `/list`
- В `/list` также появляются inline-кнопки **«Удалить #N»** для удаления в один клик
- `/add` поддерживает 3 формата:
  - `/add ДД.MM.ГГГГ ЧЧ:ММ текст` (например `25.03.2026 10:00`)
  - `/add ЧЧ:ММ текст` или `/add ЧЧ.ММ текст` (на сегодня/завтра)
  - `/add текст` (задача через 1 час)
- Также можно использовать кнопки внизу чата: **Добавить / Список / Удалить / Время / Помощь**

## Как приходят уведомления

- Уведомления приходят в Telegram-чат с пользователем, который создал задачу (поле `telegramTo`).
- По умолчанию напоминание отправляется за `15` минут до события.
- Если задан `TELEGRAM_BOT_TOKEN`, сервер шлёт уведомления напрямую через Telegram Bot API.
- `OPENCLAW_*` параметры остаются как fallback-вариант.
